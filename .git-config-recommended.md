# Recommended Git Configuration for This Project

## For Windows Users

If you're developing on Windows, you may encounter file permission issues where
`cli.py` and `bump-version.sh` always show as modified. This is because Windows
doesn't support Unix file permissions (executable bit).

**Solution**: Run this command in the repository root:

```bash
git config core.fileMode false
```

This tells Git to ignore file permission changes (mode 100755 vs 100644).

## For All Users

### Line Endings

The `.gitattributes` file enforces consistent line endings:
- Python files (`.py`) → LF (Unix)
- Shell scripts (`.sh`) → LF (Unix)
- Config files (`.yml`, `.yaml`, `.json`) → LF
- Windows batch files (`.bat`, `.cmd`) → CRLF

### Autocrlf Setting

Current setting: `core.autocrlf = input`

This means:
- On commit: Convert CRLF → LF
- On checkout: Keep LF as-is

If you encounter line ending issues:

```bash
# Clean up and re-checkout all files
git rm --cached -r .
git reset --hard
```

## Verifying Your Configuration

```bash
# Check current settings
git config --list --local | grep -E 'autocrlf|fileMode'

# Should show:
# core.autocrlf=input
# core.fileMode=false  (Windows only)
```
